To explore the localstack cluster with kubectl , you first need to authenticate to your cluster.
awslocal eks update-kubeconfig --region us-east-1 --name <cluster-name> e.g. eks-sample

Deploy sample-app to localstack ECR....Create ECR repo=> registry_url above. Build sample-app docker image with Paketo Build Pack.
pack build sample-app:v3 --builder paketobuildpacks/builder-jammy-base (https://paketo.io/docs/)

Once the Docker image is built, to be able to push it to ECR, you need to tag it with the registry URL of the ECR repo that you got from the
registry_url output:

docker tag ubuntu:focal localstack-ec2/ubuntu-focal-ami:ami-000001
awslocal ec2 describe-images --filters Name=tag:ec2_vm_manager,Values=docker


docker tag sample-app:v3 000000000000.dkr.ecr.us-east-1.localhost.localstack.cloud:4566/sample-app:v3

Next, you need to authenticate to your ECR repo, which you can do using the aws and docker CLI tools, making sure to replace the last
argument with the registry URL of your own ECR repo that you got from the registry_url output:

awslocal ecr get-login-password --region us-east-1 | docker login --username test --password-stdin 000000000000.dkr.ecr.us-east-1.localhost.localstack.cloud:4566/sample-app

Finally, you can push the Docker image to your ECR repo:
docker push 000000000000.dkr.ecr.us-east-1.localhost.localstack.cloud:4566/sample-app:v3

A LocalStack EKS LoadBalancer stuck in "Pending" usually indicates missing subnet tags, IAM permission issues, or lack of a controller to provision the LB. Ensure subnets are tagged kubernetes.io/role/internal-elb (1) or kubernetes.io/role/elb (0), and that the AWS Load Balancer Controller is installed.

Key Troubleshooting Steps for LocalStack EKS:
Verify Subnet Tags: Ensure your VPC subnets are properly tagged. For public load balancers, use kubernetes.io/role/elb = 1, and for private, kubernetes.io/role/internal-elb = 1.
Install/Check AWS Load Balancer Controller: Unlike real AWS, LocalStack requires the controller to be running in the cluster to manage ELB resources.
Check Controller Logs: If installed, check the logs of the aws-load-balancer-controller pod for errors related to subnet discovery.
Check kubectl describe svc: Run kubectl describe service <your-service-name> to see events. Often, it will state "could not find any suitable subnets".
LocalStack Configuration: Ensure LocalStack is running with SERVICES=eks,elbv2,ec2,iam and that your local setup allows for VPC and LB emulation.
Check Pod Health: Ensure the pods being targeted are running and healthy.
